I’d like you to define a clear, end-to-end **usability path** through our web app from a real user’s perspective.

Please approach this as if you are a first-time user with no internal context.

The user may provide you with a specific angle or priority they want this specific usability path to test or handle, along with reference artifacts that can be used (e.g. uploaded, etc.) for different steps.

### 1. Define the Primary User

* Who is the user? (role, goals, level of expertise)
* What are they trying to accomplish in one session?
* What assumptions do they start with?

### 2. Identify the Core Task

Choose one high-value, realistic task (e.g., “Create a new project and invite a team member”).

* State the goal in one sentence.
* Define what “success” looks like.
* Define what “failure” looks like.

### 3. Map the Ideal Usability Path

Document the step-by-step path the user would take:

* Entry point (how they arrive)
* Each screen/page they visit
* Decisions they must make
* Inputs required
* System feedback at each step
* Final confirmation/outcome

For each step, include:

* What the user expects to happen
